# Black-Box

## Мета роботи
Мета лабораторної роботи — **знайти правильну імплементацію гри "Хрестики-Нолики" за допомогою тестів.**

У репозиторії реалізовано всі компоненти гри, **окрім головного класу — двигуна гри**.  
Цей клас є нащадком `GameEngine`, який знаходиться за шляхом:  
`src/main/java/org/example/logic/api/GameEngine.java`

До проєкту додано **коментарі з описом поведінки методів**, які допоможуть зрозуміти логіку роботи гри.

---

## Етапи виконання роботи

### 1. Клонування репозиторію
1. Склонуйте репозиторій за наданим посиланням.
2. Переконайтеся, що на вашому пристрої встановлено:
    - **Java 21+**
    - **JDK 21+**
    - **Maven**
3. Саме за допомогою Maven виконуватиметься запуск програми.

**Команда для клонування:**
```git clone <посилання-на-репозиторій>```
(опційно можна додати прапорець `--depth=1`, щоб не завантажувати історію комітів)


### 2. Отримання імплементацій
1. Перейдіть за виданим вам посиланням:  
   `{link}/generate/{your-code}`
2. Вам буде надано **архів (.zip)** із 1000 імплементаціями — 999 неправильних і лише **одна правильна**.
3. Розпакуйте архів у папку:  
   `src/main/java/org/example/impl`  
   (у цій директорії мають з’явитися `.java` файли)


### 3. Пошук правильної імплементації
1. Відкрийте файл тестів за шляхом:  
   `src/test/java/org/example/logic/api/GameEngineTest.java`
2. Реалізуйте тести, які дозволять **визначити правильну імплементацію**.
3. Тести потрібно писати на **батьківський клас `GameEngine`**.  
   Усі потрібні методи задокументовані в коді.
4. Не змінюйте інші частини проєкту — це може порушити роботу тест-раннера.

## Важливі моменти

### 1. Клонування
Якщо Git не встановлено — обов’язково інсталюйте його.  
Без цього неможливо буде клонувати проєкт і виконати лабораторну.

**Команда:** `git clone <посилання-на-репозиторій>`

---

### 2. Тест-раннер
Для запуску тестів усіх імплементацій використайте команду (у корені проєкту):

```mvn test-compile```

Після запуску:
- у **консолі** з’являться результати тестів для кожного класу;
- у **корені проєкту** створиться файл `tests_summary.txt`,  
  де будуть записані всі імплементації, які пройшли ваші тести.

**Завдання:** зробити так, щоб у цьому файлі залишилася **лише одна** імплементація — правильна.
